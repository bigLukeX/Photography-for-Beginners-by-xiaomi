# Project Overview

这份文档面向“新接手本项目的人类或 AI”。

它不替代 `README`，而是更偏向实际接手时需要的上下文。

## 1. 这个项目在做什么

项目目标不是单纯做一个摄影博客，而是做一套有课程感的摄影电子教程。

特点：

- 以 `小米 15 Ultra` 为现实起点
- 内容面向摄影初学者
- 不把用户锁死在“手机摄影技巧”
- 默认考虑未来迁移到相机
- 每章都有练习和点评思路
- 每个作业有 20 分钟核心版、完整挑战和点评后重拍三层
- 真实教学样张都有作者、许可和来源归档

## 2. 为什么用现在这套技术

选择 `Astro + Starlight + MDX` 的原因：

- `Starlight` 很适合文档站和课程站
- `MDX` 能让教程内容和交互式示意组件共存
- 最终是静态站，适合 GitHub Pages
- 维护成本低，长期迭代友好

## 3. 内容策略

这套教程优先解决的是“建立摄影判断力”，不是“堆设备知识”。

所以推荐顺序被故意设计成：

1. 相机 App 起步
2. 第 1-3 章：主体、构图、光线
3. 焦距专题、第 4-5 章、曝光、色彩、第 10 章
4. 第 6-9 章：题材专项
5. 第 11 章、选片、后期、组图、第 12 章
6. 先第 17 章，再第 13-16、18-19 章

不要把侧边栏改回纯数字顺序。第 10 章前置是为了先学参数再进题材，第 17 章前置是为了先有风格地图再学单项效果。

## 6. 章节扩写模板建议

后续正式章节建议尽量按这个节奏写：

1. 开场：这一章为什么重要
2. 学完以后能做到什么
3. 核心概念拆解
4. 手机上的实战方法
5. 常见错误 vs 推荐做法
6. 练习题
7. 自检标准
8. 点评标准
9. 换成相机如何迁移
10. 一句重点总结

## 7. 设计与视觉方向

当前站点不是“极简工具文档风”，而是偏暖色、带一点教程气质和轻示意动画的方向。

已建立的视觉特征：

- 暖色主色调
- 柔和卡片容器
- 渐变背景
- 适量示意图和轻动画

新增内容时应保持这个方向，不要突然切成非常冷硬或完全系统默认风格。

## 8. 响应式要求

本项目默认需要支持：

- 手机阅读
- 平板阅读
- 桌面阅读

新增组件时，至少考虑：

- 小屏时是否应该单列
- 容器高度是否会挤爆
- 标签文字是否过长
- 表格是否会在手机上难以阅读

## 9. 目前最值得继续推进的工作

高优先级：

1. 用用户授权的作业补“初拍 → 点评 → 重拍”案例
2. 给第 19 章补带 EXIF 的原创参数对照图
3. 补小米 15 Ultra 实机界面截图与版本说明

中优先级：

1. 给第 11 章补更细的可撤销调色流程
2. 给高级篇补同一底片的不同后期方向
3. 扩展真实样张，但必须同步图片许可归档

低优先级但后面会有价值：

1. 补更多原创插图
2. 做章节间风格更统一的视觉锚点
3. 给不同风格建立可打印的拍摄清单

## 10. 接手时的工作流建议

如果要继续开发，建议这样做：

1. 先读 `README` 和 `AGENTS.md`
2. 选定一个章节，不要一次改很多页
3. 尽量复用已有组件
4. 新增组件时补响应式
5. 每次都跑 `npm run build`
6. 若改动是布局类，再用浏览器验证手机和平板

## 11. 特别注意

- 首页和 404 的链接要小心 `GitHub Pages` 项目子路径问题
- `Starlight` 会自动渲染页面标题，不要重复写一级标题
- 教程写作要避免“器材优越感”
- 这套项目的价值在于“可学、可练、可点评”，不是只堆内容长度
- 新增第三方图片时必须同步 `docs/IMAGE_CREDITS.md`，并在页面显示作者、许可和原始链接
- `PracticeCard` 默认自动显示三层作业提示；仅说明页等特殊场景使用 `tiered={false}`
