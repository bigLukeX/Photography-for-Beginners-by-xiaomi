# Project Overview

这份文档面向“新接手本项目的人类或 AI”。

它不替代 `README`，而是更偏向实际接手时需要的上下文。

## 1. 这个项目在做什么

项目目标不是单纯做一个摄影博客，而是做一套有课程感的摄影电子教程。

特点：

- 以 `小米 15 Ultra` 为现实起点
- 内容面向摄影初学者
- 不把用户锁死在“手机摄影技巧”
- 默认考虑未来迁移到相机
- 每章都有练习和点评思路

## 2. 为什么用现在这套技术

选择 `Astro + Starlight + MDX` 的原因：

- `Starlight` 很适合文档站和课程站
- `MDX` 能让教程内容和交互式示意组件共存
- 最终是静态站，适合 GitHub Pages
- 维护成本低，长期迭代友好

## 3. 内容策略

这套教程优先解决的是“建立摄影判断力”，不是“堆设备知识”。

所以它的内容顺序被故意设计成：

1. 清晰度与主体
2. 构图
3. 光线
4. 焦段与镜头
5. 摄影底层概念
6. 题材专项
7. 参数控制
8. 后期与选片
9. 拍摄流程
10. 高级创作：粗粝、模糊、拖影、破碎取景、梦境感
11. 参数工具箱：快门、ISO、白平衡、曝光补偿、对焦和风格配方

## 6. 章节扩写模板建议

后续正式章节建议尽量按这个节奏写：

1. 开场：这一章为什么重要
2. 学完以后能做到什么
3. 核心概念拆解
4. 手机上的实战方法
5. 常见错误 vs 推荐做法
6. 练习题
7. 自检标准
8. 点评标准
9. 换成相机如何迁移
10. 一句重点总结

## 7. 设计与视觉方向

当前站点不是“极简工具文档风”，而是偏暖色、带一点教程气质和轻示意动画的方向。

已建立的视觉特征：

- 暖色主色调
- 柔和卡片容器
- 渐变背景
- 适量示意图和轻动画

新增内容时应保持这个方向，不要突然切成非常冷硬或完全系统默认风格。

## 8. 响应式要求

本项目默认需要支持：

- 手机阅读
- 平板阅读
- 桌面阅读

新增组件时，至少考虑：

- 小屏时是否应该单列
- 容器高度是否会挤爆
- 标签文字是否过长
- 表格是否会在手机上难以阅读

## 9. 目前最值得继续推进的工作

高优先级：

1. 给第 19 章补真实参数对照图
2. 给高级篇补真实作业示例和对照图
3. 增加作品页，用来整理用户后续拍摄结果

中优先级：

1. 给每章补更多手机设置截图
2. 给第 11 章补更细的调色流程
3. 给高级篇补更多后期参数建议

低优先级但后面会有价值：

1. 补更多原创插图
2. 做章节间风格更统一的视觉锚点
3. 给不同风格建立可打印的拍摄清单

## 10. 接手时的工作流建议

如果要继续开发，建议这样做：

1. 先读 `README` 和 `AGENTS.md`
2. 选定一个章节，不要一次改很多页
3. 尽量复用已有组件
4. 新增组件时补响应式
5. 每次都跑 `npm run build`
6. 若改动是布局类，再用浏览器验证手机和平板

## 11. 特别注意

- 首页和 404 的链接要小心 `GitHub Pages` 项目子路径问题
- `Starlight` 会自动渲染页面标题，不要重复写一级标题
- 教程写作要避免“器材优越感”
- 这套项目的价值在于“可学、可练、可点评”，不是只堆内容长度
